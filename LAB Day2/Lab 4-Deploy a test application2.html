Port Forwarding

In this section you will verify the ability to access applications remotely using port forwarding.

Retrieve the full name of the nginx pod:

root@master:~# POD_NAME=$(kubectl get pods -l run=nginx -o jsonpath="{.items[0].metadata.name}")

Forward port 8080 on your local machine to port 80 of the nginx pod:

root@master:~# kubectl port-forward $POD_NAME 8080:80

Output

Forwarding from 127.0.0.1:8080 -> 80
Forwarding from [::1]:8080 -> 80

In a new terminal make an HTTP request using the forwarding address:

ksadmin@master:~$ curl --head http://127.0.0.1:8080

Output

HTTP/1.1 200 OK
Server: nginx/1.15.12
Date: Sun, 19 May 2019 12:19:06 GMT
Content-Type: text/html
Content-Length: 612
Last-Modified: Tue, 16 Apr 2019 13:08:19 GMT
Connection: keep-alive
ETag: "5cb5d3c3-264"
Accept-Ranges: bytes

Switch back to the previous terminal and stop the port forwarding to the nginx pod:

root@master:~# kubectl port-forward $POD_NAME 8080:80
Forwarding from 127.0.0.1:8080 -> 80
Forwarding from [::1]:8080 -> 80

Handling connection for 8080

Services

In this section you will verify the ability to expose applications using a Service.

Expose the nginx deployment using a NodePort service:

root@master:~# kubectl expose deployment nginx --port 80 --type NodePort

Output 

service/nginx exposed

Retrieve the node port assigned to the nginx service:

root@master:~#  NODE_PORT=$(kubectl get svc nginx \
>   --output=jsonpath='{range .spec.ports[0]}{.nodePort}')
root@master:~# echo $NODE_PORT

Output

30766

Make an HTTP request using the external IP address and the nginx node port:

root@master:~# curl -I http://10.40.41.169:${NODE_PORT}

Output

HTTP/1.1 200 OK
Server: nginx/1.15.12
Date: Sun, 19 May 2019 12:30:32 GMT
Content-Type: text/html
Content-Length: 612
Last-Modified: Tue, 16 Apr 2019 13:08:19 GMT
Connection: keep-alive
ETag: "5cb5d3c3-264"
Accept-Ranges: bytes
